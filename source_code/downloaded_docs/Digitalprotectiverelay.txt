     <p>
      In utility and industrial
      <a href="/wiki/Electric_power_transmission" title="Electric power transmission">
       electric power transmission
      </a>
      and
      <a class="mw-redirect" href="/wiki/Electricity_distribution" title="Electricity distribution">
       distribution
      </a>
      systems, a
      <b>
       digital protective relay
      </b>
      is a computer-based system with software-based protection
      <a href="/wiki/Algorithm" title="Algorithm">
       algorithms
      </a>
      for the detection of electrical
      <a href="/wiki/Fault_(power_engineering)" title="Fault (power engineering)">
       faults
      </a>
      .
      <sup class="reference" id="cite_ref-1">
       <a href="#cite_note-1">
        [1]
       </a>
      </sup>
      Such relays are also termed as
      <a href="/wiki/Microprocessor" title="Microprocessor">
       microprocessor
      </a>
      type protective relays. They are functional replacements for electro-mechanical
      <a href="/wiki/Protective_relay" title="Protective relay">
       protective relays
      </a>
      and may include many protection functions in one unit, as well as providing metering, communication, and self-test functions.
     </p>

     <h2>
      <span class="mw-headline" id="Description_and_definition">
       Description and definition
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Digital_protective_relay&amp;action=edit&amp;section=1" title="Edit section: Description and definition">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      The digital protective
      <a href="/wiki/Relay" title="Relay">
       relay
      </a>
      is a
      <a href="/wiki/Protective_relay" title="Protective relay">
       protective relay
      </a>
      that uses a
      <a href="/wiki/Microprocessor" title="Microprocessor">
       microprocessor
      </a>
      to analyze power system voltages, currents or other process quantities for the purpose of detection of faults in an electric power system or industrial process system. A digital protective relay may also be called a "numeric protective relay".
     </p>
     <h3>
      <span class="mw-headline" id="Input_processing">
       Input processing
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Digital_protective_relay&amp;action=edit&amp;section=2" title="Edit section: Input processing">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <p>
      Low
      <a href="/wiki/Voltage" title="Voltage">
       voltage
      </a>
      and low
      <a href="/wiki/Electric_current" title="Electric current">
       current
      </a>
      signals (i.e., at the secondary of a
      <a href="/wiki/Transformer_types#Voltage_transformers" title="Transformer types">
       voltage transformers
      </a>
      and
      <a href="/wiki/Current_transformer" title="Current transformer">
       current transformers
      </a>
      ) are brought into a
      <a class="mw-redirect" href="/wiki/Low_pass_filter" title="Low pass filter">
       low pass filter
      </a>
      that removes
      <a href="/wiki/Frequency" title="Frequency">
       frequency
      </a>
      content above about 1/3 of the
      <a class="mw-redirect" href="/wiki/Sampling_frequency" title="Sampling frequency">
       sampling frequency
      </a>
      (a relay
      <a class="mw-redirect" href="/wiki/A/D_converter" title="A/D converter">
       A/D converter
      </a>
      needs to sample faster than twice per cycle of the highest frequency that it is to monitor). The
      <a href="/wiki/Alternating_current" title="Alternating current">
       AC
      </a>
      signal is then sampled by the relay's analog to digital converter from 4 to 64 (varies by relay) samples per power system cycle. As a minimum, magnitude of the incoming quantity, commonly using
      <a href="/wiki/Fourier_transform" title="Fourier transform">
       Fourier transform
      </a>
      concepts (
      <a href="/wiki/Root_mean_square" title="Root mean square">
       RMS
      </a>
      and some form of averaging) would be used in a simple relay function. More advanced analysis can be used to determine
      <a href="/wiki/Phase_angle" title="Phase angle">
       phase angles
      </a>
      ,
      <a href="/wiki/AC_power" title="AC power">
       power
      </a>
      ,
      <a class="mw-redirect" href="/wiki/Reactive_power" title="Reactive power">
       reactive power
      </a>
      ,
      <a href="/wiki/Electrical_impedance" title="Electrical impedance">
       impedance
      </a>
      ,
      <a href="/wiki/Waveform" title="Waveform">
       waveform
      </a>
      <a href="/wiki/Distortion" title="Distortion">
       distortion
      </a>
      , and other
      <a href="/wiki/Complex_system" title="Complex system">
       complex
      </a>
      quantities.
     </p>
     <p>
      Only the fundamental component is needed for most protection algorithms, unless a high speed algorithm is used that uses subcycle data to monitor for fast changing issues. The sampled data is then passed through a low pass filter that numerically removes the frequency content that is above the
      <a href="/wiki/Fundamental_frequency" title="Fundamental frequency">
       fundamental frequency
      </a>
      of interest (i.e., nominal system frequency), and uses Fourier transform algorithms to extract the fundamental frequency magnitude and angle.
     </p>
     <h3>
      <span class="mw-headline" id="Logic_processing">
       Logic processing
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Digital_protective_relay&amp;action=edit&amp;section=3" title="Edit section: Logic processing">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <p>
      The relay analyzes the resultant A/D converter outputs to determine if action is required under its protection algorithm(s). Protection algorithms are a set of logic equations in part designed by the protection engineer, and in part designed by the relay manufacturer. The relay is capable of applying advanced logic. It is capable of analyzing whether the relay should trip or restrain from tripping based on parameters set by the user, compared against many functions of its
      <a href="/wiki/Analog_signal" title="Analog signal">
       analogue
      </a>
      inputs, relay contact inputs, timing and order of event sequences.
     </p>
     <p>
      If a fault condition is detected, output contacts operate to trip the associated circuit breaker(s).
     </p>
     <h3>
      <span class="mw-headline" id="Parameter_setting">
       Parameter setting
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Digital_protective_relay&amp;action=edit&amp;section=4" title="Edit section: Parameter setting">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <p>
      The logic is user-configurable and can vary from simply changing front panel switches or moving of
      <a class="mw-redirect" href="/wiki/Circuit_board" title="Circuit board">
       circuit board
      </a>
      jumpers to accessing the relay's internal parameter setting webpage via communications link on another computer hundreds of kilometres away.
     </p>
     <p>
      The relay may have an extensive collection of settings, beyond what can be entered via front panel knobs and dials, and these settings are transferred to the relay via an interface with a PC (
      <a href="/wiki/Personal_computer" title="Personal computer">
       personal computer
      </a>
      ), and this same PC interface may be used to collect event reports from the relay.
     </p>
     <h3>
      <span class="mw-headline" id="Event_recording">
       Event recording
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Digital_protective_relay&amp;action=edit&amp;section=5" title="Edit section: Event recording">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <p>
      In some relays, a short history of the entire sampled data is kept for oscillographic records. The event recording would include some means for the user to see the timing of key logic decisions, relay I/O (input/output) changes, and see, in an
      <a href="/wiki/Oscillograph" title="Oscillograph">
       oscillographic
      </a>
      fashion, at least the fundamental component of the incoming analogue parameters.
     </p>
     <h3>
      <span class="mw-headline" id="Data_display">
       Data display
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Digital_protective_relay&amp;action=edit&amp;section=6" title="Edit section: Data display">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h3>
     <p>
      Digital/numerical relays provide a front panel display, or display on a terminal through a communication interface. This is used to display relay settings and real-time current/voltage values, etc.
     </p>
     <p>
      More complex digital relays will have metering and communication protocol ports, allowing the relay to become an element in a
      <a href="/wiki/SCADA" title="SCADA">
       SCADA
      </a>
      system. Communication ports may include
      <a class="mw-redirect" href="/wiki/RS232" title="RS232">
       RS232
      </a>
      /
      <a class="mw-redirect" href="/wiki/RS485" title="RS485">
       RS485
      </a>
      or
      <a href="/wiki/Ethernet" title="Ethernet">
       Ethernet
      </a>
      (copper or fibre-optic). Communication languages may include
      <a href="/wiki/Modbus" title="Modbus">
       Modbus
      </a>
      ,
      <a href="/wiki/DNP3" title="DNP3">
       DNP3
      </a>
      or
      <a class="mw-redirect" href="/wiki/IEC61850" title="IEC61850">
       IEC61850
      </a>
      protocols.
     </p>
     <h2>
      <span class="mw-headline" id="Comparison_with_other_types">
       Comparison with other types
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Digital_protective_relay&amp;action=edit&amp;section=7" title="Edit section: Comparison with other types">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <div class="thumb tright">
      <div class="thumbinner" style="width:222px;">
       <a class="image" href="/wiki/File:Protective_Relays_Hydroelectric_Station.JPG">
        <img alt="" class="thumbimage" data-file-height="1200" data-file-width="1600" height="165" src="//upload.wikimedia.org/wikipedia/commons/thumb/c/c1/Protective_Relays_Hydroelectric_Station.JPG/220px-Protective_Relays_Hydroelectric_Station.JPG" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/c/c1/Protective_Relays_Hydroelectric_Station.JPG/330px-Protective_Relays_Hydroelectric_Station.JPG 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/c/c1/Protective_Relays_Hydroelectric_Station.JPG/440px-Protective_Relays_Hydroelectric_Station.JPG 2x" width="220"/>
       </a>
       <div class="thumbcaption">
        <div class="magnify">
         <a class="internal" href="/wiki/File:Protective_Relays_Hydroelectric_Station.JPG" title="Enlarge">
         </a>
        </div>
        Electromechanical protective relays at a hydroelectric generation station
       </div>
      </div>
     </div>
     <p>
      By contrast, an electromechanical
      <a href="/wiki/Protective_relay" title="Protective relay">
       protective relay
      </a>
      converts the voltages and currents to magnetic and electric forces and torques that press against spring tensions in the relay. The tension of the spring and taps on the electromagnetic coils in the relay are the main processes by which a user sets such a relay.
     </p>
     <p>
      In a
      <a class="mw-redirect" href="/wiki/Solid_state_relay" title="Solid state relay">
       solid state relay
      </a>
      , the incoming voltage and current wave-forms are monitored by analog circuits, not recorded or digitized. The analog values are compared to settings made by the user via
      <a href="/wiki/Potentiometer" title="Potentiometer">
       potentiometers
      </a>
      in the relay, and in some case, taps on transformers.
     </p>
     <p>
      In some
      <a class="mw-redirect" href="/wiki/Solid_state_relay" title="Solid state relay">
       solid state relays
      </a>
      , a simple
      <a href="/wiki/Microprocessor" title="Microprocessor">
       microprocessor
      </a>
      does some of the relay logic, but the logic is fixed and simple. For instance, in some time overcurrent solid state relays, the incoming AC current is first converted into a small signal AC value, then the AC is fed into a
      <a href="/wiki/Rectifier" title="Rectifier">
       rectifier
      </a>
      and filter that converts the
      <a href="/wiki/Alternating_current" title="Alternating current">
       AC
      </a>
      to a
      <a href="/wiki/Direct_current" title="Direct current">
       DC
      </a>
      value proportionate to the AC waveform. An
      <a class="mw-redirect" href="/wiki/Op-amp" title="Op-amp">
       op-amp
      </a>
      and comparator is used to create a DC that rises when a trip point is reached. Then a relatively simple microprocessor does a slow speed A/D conversion of the DC signal, integrates the results to create the time-overcurrent curve response, and trips when the integration rises above a set-point. Though this relay has a microprocessor, it lacks the attributes of a digital/numeric relay, and hence the term "microprocessor relay" is not a clear term.
     </p>
     <h2>
      <span class="mw-headline" id="History">
       History
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Digital_protective_relay&amp;action=edit&amp;section=8" title="Edit section: History">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      The digital/numeric relay was introduced in the early 1980s, with
      <a class="mw-redirect" href="/wiki/AREVA" title="AREVA">
       AREVA
      </a>
      ,
      <a href="/wiki/ABB_Group" title="ABB Group">
       ABB Group
      </a>
      's forerunners and
      <a href="/wiki/Schweitzer_Engineering_Laboratories" title="Schweitzer Engineering Laboratories">
       SEL
      </a>
      making some of the early market advances in the arena, but the arena has become crowded today with many manufacturers. In transmission line and generator protection, by the mid-1990s the digital relay had nearly replaced the solid state and electro-mechanical relay in new construction. In distribution applications, the replacement by the digital relay proceeded a bit more slowly. While the great majority of feeder relays in new applications today are digital, the solid state relay still sees some use where simplicity of the application allows for simpler relays, which allows one to avoid the complexity of digital relays.
     </p>
     <h2>
      <span class="mw-headline" id="Protective_element_types">
       Protective element types
      </span>
      <span class="mw-editsection">
       <span class="mw-editsection-bracket">
        [
       </span>
       <a href="/w/index.php?title=Digital_protective_relay&amp;action=edit&amp;section=9" title="Edit section: Protective element types">
        edit
       </a>
       <span class="mw-editsection-bracket">
        ]
       </span>
      </span>
     </h2>
     <p>
      Protective elements refer to the overall logic surrounding the electrical condition that is being monitored. For instance, a differential element refers to the logic required to monitor two (or more) currents, find their difference, and trip if the difference is beyond certain parameters. The term element and function are quite interchangeable in many instances.
     </p>
     <p>
      For simplicity on one-line diagrams, the protection function is usually identified by an
      <a class="mw-redirect" href="/wiki/ANSI" title="ANSI">
       ANSI
      </a>
      device number. In the era of electromechanical and solid state relays, any one relay could implement only one or two protective functions, so a complete protection system may have many relays on its panel. In a digital/numeric relay, many functions are implemented by the microprocessor programming. Any one numeric relay may implement one or all of these functions.
     </p>
     <p>
      A listing of device numbers is found at
      <a class="mw-redirect" href="/wiki/ANSI_Device_Numbers" title="ANSI Device Numbers">
       ANSI Device Numbers
      </a>
      . A summary of some common device numbers seen in digital relays is:
     </p>
     <ul>
      <li>
       11 – Multi-function Device
      </li>
      <li>
       21 – Impedance
      </li>
      <li>
       24 – Volts/Hz
      </li>
      <li>
       25 – Synchronizing
      </li>
      <li>
       27 – Under Voltage
      </li>
      <li>
       32 – Directional Power Element
      </li>
      <li>
       46 – Negative Sequence Current
      </li>
      <li>
       40 – Loss of Excitation
      </li>
      <li>
       47 – Negative Sequence Voltage
      </li>
      <li>
       50 – Instantaneous Overcurrent (N for neutral, G for ground current)
      </li>
      <li>
       51 – Inverse Time Overcurrent (N for neutral, G from ground current)
      </li>
      <li>
       59 – Over Voltage
      </li>
      <li>
       62 – Timer
      </li>
      <li>
       64 – Ground Fault (64F = Field Ground, 64G = Generator Ground)
      </li>
      <li>
       67 – Directional Over Current (typically controls a 50/51 element)
      </li>
      <li>
       79 – Reclosing Relay
      </li>
      <li>
       81 – Under/Over Frequency
      </li>
      <li>
       86 – Lockout Relay / Trip Circuit Supervision
      </li>
      <li>
       87 – Current Differential (87L=transmission line diff; 87T=transformer diff; 87G=generator diff)
      </li>
     </ul>
